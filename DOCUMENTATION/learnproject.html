5. üîÑ Code Flow & Architecture

5.1 Application Startup Flow

Step 1: Load environment variables (.env file)
load_dotenv()
Step 2: Create Flask app using factory pattern
app = create_app('development')
Step 3: Load configuration (config.py)
app.config.from_object(DevelopmentConfig)
Step 4: Initialize extensions (SQLAlchemy, Flask-Login, SocketIO, etc.)
db.init_app(app) login_manager.init_app(app) socketio.init_app(app)
Step 5: Register blueprints (routes)
app.register_blueprint(main_bp) app.register_blueprint(auth_bp, url_prefix='/auth') app.register_blueprint(service_bp, url_prefix='/service')
Step 6: Create database tables
db.create_all()
Step 7: Run migrations (add missing columns)
run_migrations(app)
Step 8: Create default admin user
create_default_admin(app)
Step 9: Start server with SocketIO
socketio.run(app, host='0.0.0.0', port=5000)

5.2 Request-Response Flow (Example: Place Order)

1. User Action: Client clicks "Place Order" button
2. Route: POST /service/<service_id>/order
@service_bp.route('/<int:service_id>/order', methods=['POST'])
3. Authentication Check: @login_required decorator
4. Validation: Check wallet balance, service availability
5. Payment Processing:
  • Deduct from buyer wallet (100%)
  • Credit to seller wallet (90%)
  • Credit to admin wallet (10% platform fee)
6. Database Operations:
  • Create Order record
  • Create Transaction records (3 entries)
  • Update wallet balances
7. Notifications:
  • Send notification to seller
  • Send email to buyer and seller
8. Response: Redirect to order detail page

6. ‚ö° Key Features Implementation

6.1 User Authentication

Files: routes.py (auth_bp), models.py (User model)
Features:
  • Registration with email verification
  • Login with password hashing (Werkzeug)
  • Google OAuth integration (Authlib)
  • Session management (Flask-Login)
  • Role-based access (client, provider, admin)

6.2 Wallet System

Files: payment_system.py, routes.py (user_bp)
Payment Flow:
  1. Add Money: Client/Provider adds money via card/UPI
  2. Order Payment:
    • Buyer pays 100% from wallet
    • 90% credited to seller wallet
    • 10% platform fee to admin wallet
  3. Refund: On order rejection, full refund to buyer + reverse seller/admin credits
Transaction Types:
  • Credit: Money added (green icon)
  • Debit: Money deducted (red icon)

6.3 Order Management

Files: routes.py, managers.py (OrderManager)
Order Lifecycle:
  1. Pending: Order placed, waiting for seller acceptance
  2. In Progress: Seller accepted, work in progress
  3. Completed: Work delivered, certificate issued
  4. Cancelled: Rejected by seller (reason mandatory)
Key Features:
  • Mandatory rejection reason when seller rejects
  • Automatic refund on rejection
  • Real-time chat between buyer and seller
  • Order tracking and status updates

6.4 Certificate System

Files: certificate_generator.py, routes.py
Process:
  1. Order marked as completed by seller
  2. System auto-generates PDF certificate
  3. Certificate stored in database with unique ID (CERT-XXXXXX)
  4. Buyer receives notification with download link
  5. Certificate can be verified using cert_id
Certificate Contains:
  • Student name
  • Skill/service name
  • Provider name
  • Issue date
  • Unique certificate ID
  • QR code for verification

6.5 Real-time Chat

Files: events.py, routes.py, templates/user/order_detail.html
Technology: Flask-SocketIO (WebSocket)
Features:
  • Real-time messaging between buyer and seller
  • Chat linked to specific order
  • Message history stored in database
  • Online/offline status indicators
  • Hide/unhide chat conversations

6.6 Booking & Availability System

Files: routes.py (availability_bp), models.py
How it Works:
  1. Provider creates availability slots (date + time)
  2. Client books a slot when placing order
  3. Provider approves/rejects booking request
  4. Once approved, order can be accepted
  5. Slot marked as booked

6.7 AskVera AI Chatbot

Files: chat_manager.py, routes_chat.py
Technology: Groq AI (LLaMA 3.3 70B model)
Features:
  • Context-aware responses based on user role
  • Different suggestions for guest, client, provider, admin
  • Knows about all platform features
  • Provides help with wallet, orders, certificates, etc.
  • Real-time responses

6.8 Admin Features

Files: routes.py (admin_bp), templates/admin/*
Admin Can:
  • View all users, services, orders
  • Approve/reject service listings
  • Manage categories
  • View platform statistics
  • Receive 10% platform fee from all orders
Admin Restrictions:
  • Cannot add money to wallet (only receives fees)
  • Cannot place orders
  • "Add Money" button hidden
  • "My Orders" menu hidden

6.9 Email Notifications

Files: email_utils.py, templates/emails/*
Email Types:
  • Welcome email on registration
  • Order placed (to buyer and seller)
  • Order accepted (to buyer and seller)
  • Order completed (to buyer and seller)
  • Order rejected (to buyer with reason)
  • Booking confirmation/rejection
